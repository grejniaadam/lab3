- Use best Python programming practices.
- Use the current repository context and file structure.

## Requirements to follow
- Python 3.9+.
- Apply PEP 8 formatting and linting rules.

- Keep modules under 500 lines and functions under 50 lines.

- Keep `requirements.txt` updated if new dependencies are needed.

## Security and configuration
- Never hard-code credentials or secrets.

- Keep configuration secure and avoid exposing secrets in generated examples.

## Git and workflow
- Use conventional commit messages.
- Prefer branch names like `feature/*`, `bugfix/*`, or `docs/*`.
- Include test coverage and behavior description in PRs.

## Desired output
- Generate Python code or project files only when asked.
- Provide code in valid Python syntax.
- If a task requires a file, create it with a proper file path.
- If the task is ambiguous, ask for clarifying details.
